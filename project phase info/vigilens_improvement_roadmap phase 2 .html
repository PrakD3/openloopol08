Temporal displacement agent
A dedicated 4th agent that cross-references EXIF timestamps, YouTube upload dates, and Wayback Machine archives to catch real footage used in the wrong time context — the type that defeats all deepfake detectors.
critical gap medium effort
Why: 51% of all disaster misinfo is false context (Turkey-Syria earthquake study). Your deepfake detector can't catch a real 2015 flood reposted as 2025. This agent is the only one that can.

SSE streaming for live agent progress

Hash-only storage (privacy architecture)
Never store raw video — store only perceptual hashes and metadata fingerprints. Two benefits: instant verdicts on previously analyzed videos (network effect), and the trust layer that makes NGOs and journalists comfortable using the platform.
trust + network effect low effort
Why: Disaster footage often contains sensitive content. Any organization that handles humanitarian data won't use a platform that stores videos. Hash-only also enables a free "instant match" lookup for repeat videos.

Audio deepfake detection
Route Whisper's audio features + transcript to a voice authenticity model. ElevenLabs-style cloned voices on fake news clips are the fastest-growing misinfo vector. You already have the audio — just need a classifier on top.
fastest growing threat medium effort
Why: MarketsandMarkets reports that AI audio/voice detection is the fastest-growing segment in the entire AI detector market (2025-2030). Adding it now makes Vigilens future-proof and differentiates from every competitor.

Newsroom API + webhook integration
A documented REST API and webhook system so newsrooms can integrate Vigilens into their CMS workflows. When a journalist pastes a URL into their draft, Vigilens verdict appears inline. Target: AFP, Reuters tools, Indian news agencies.
B2B revenue path medium effort
Why: This is the commercial monetization path. Enterprise newsroom API licenses are the revenue model that sustains the free NGO tier. AFP already has Vera.ai — positioning Vigilens as the disaster-specialized alternative is a clear wedge.

Reputation-weighted community voting
Simple karma layer on the community feed: verified journalists and NGO accounts carry 10× vote weight. Anonymous votes are still counted but down-weighted. Protects verdicts from coordinated manipulation during viral events.
sybil resistance medium effort
Why: Coordinated misinfo campaigns specifically target fact-checking community features during disasters. Without reputation weighting, a community vote becomes a bot farm target. Supabase is already in your stack — just needs a role column.

PWA / mobile-first experience
Convert the frontend into a Progressive Web App with "Add to Home Screen" support. On-ground journalists and aid workers verify footage on phones, not laptops. Android share-sheet integration so users can share videos directly from WhatsApp or browser into Vigilens.
field usability low effort
Why: Next.js makes PWA conversion trivial with next-pwa. The target user in India, Africa, and Southeast Asia is primarily mobile. Desktop-only severely limits real-world adoption in the exact markets that need this most.

Perceptual hash lookup database
Every analyzed video's perceptual hash gets stored. Future submissions are instantly matched against this database — giving instant verdicts for viral content and building a growing "disaster footage immunity" index over time.
compounding network effect low effort
Why: Viral misinfo videos get submitted hundreds of times. After the first analysis, every subsequent submission should return in milliseconds, not 60 seconds. This is also the foundation for federating verdicts across multiple Vigilens nodes.

WhatsApp bot integration
A dedicated WhatsApp number that users forward suspicious videos to, receiving a verdict reply with the shareable card link. In India, WhatsApp is the primary misinfo vector — not Twitter. This is where the real volume lives.
India market unlock high effort
Why: WhatsApp Business API + Twilio makes this technically feasible, but requires production backend stability and WhatsApp partner approval. The reach payoff is enormous — India's 500M WhatsApp users forwarding disaster clips is the exact target population.

Browser extension for inline verification
A Chrome/Firefox extension that adds a "Verify with Vigilens" button directly on Twitter/X, Facebook, and YouTube video embeds. One click — no URL copying, no tab switching. Verdict appears as a tooltip overlay.
zero-friction adoption high effort
Why: The single biggest barrier to fact-checking is switching context. If the verify button is already on the page, usage multiplies. InVid has proven this model — but they don't do video-native multi-agent analysis.

GIS / satellite location validation
Cross-reference the claimed location in video metadata or captions with satellite imagery of the actual event. Flags cases where "this Kerala flood footage shows terrain inconsistent with Kerala geography" — a level of analysis no current tool attempts.
unique differentiator high effort
Why: This is the category-defining feature that no competitor has even attempted. It requires Sentinel/Landsat satellite APIs + a vision model for terrain matching, but the headline "AI verifies disaster location against satellite data" is a PR and grant magnet.
